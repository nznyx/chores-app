# Documentation

This directory contains project documentation for the Chores App.

## Contents

### `diagrams/`
Visual documentation - architecture diagrams, user flow diagrams, entity relationships, etc.

### `requirements/`
Structured requirements for the project, split into two categories:

- [FR.md](requirements/FR.md) — Functional requirements. What the app must do: feature specs, user scenarios, business rules, permission logic, and so on.
- [NFR.md](requirements/NFR.md) — Non-functional requirements. How the app must behave: performance, scalability, security, self-hosting constraints, platform support, etc.
